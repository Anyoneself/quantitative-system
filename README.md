# Quantitative-System

Quantitative-System 是一个面向 A 股单股分析和全市场扫描的量化辅助决策系统。

项目当前基于公开日线行情构建多层技术分析：价格、成交量、均线、年线、量价形态、机器学习相似样本、缠论结构和持仓卖出风险。用户输入股票代码和机器学习算法后，系统会实时抓取该股票的最新日线数据，计算短中长期趋势指标，识别结构位置，并输出适合买入评分和适合卖出评分。系统统一使用免登录公开行情接口。

这个项目不是自动交易系统，也不承诺盈利。它的定位是一个“股票技术分析助手”：帮助用户更清晰地看到当前股票的趋势、量能、年线位置、缠论结构、买入风险、卖出风险、机器学习预测结果，以及全 A 市场里相对更值得观察的候选股票。

## 项目目标

- 输入一个股票代码，输出适合买入评分、适合卖出评分和对应依据。
- 前端支持抓取全 A 股票列表，扫描过程中实时更新当前 Top 10，整轮完成后停止，下一轮需要手动重新启动。
- 给出机器学习预测概率和缠论结构调整，而不是只给一句模糊结论。
- 给出详细理由、风险提示、缠论结构辅助、持仓卖出判断和后续观察点。
- 每次分析都实时抓取最新行情数据，不依赖本地缓存。
- 保持实现简单，避免冗余设计。

## 适用场景

- 想快速查看一只股票当前是否值得买入观察，或持仓后是否需要减仓/止盈/止损。
- 想了解一只股票最近量价状态、年线位置，以及分型、笔、中枢等结构位置。
- 想用简单机器学习模型辅助判断下一个交易日上涨概率。
- 想看到买入依据、卖出依据、风险解释和后续观察点。

## 当前功能

- 每次从公开行情接口抓取最新日线数据。
- 自动识别股票名称，例如 `601988` 对应 `中国银行`。
- 计算最近 21 个交易日的价格、成交量、均线、涨跌幅、成交额和流动性指标。
- 计算年线 MA250、年线方向、是否站上年线、是否缩量回踩年线等长期趋势指标。
- 识别涨停缩量、放量突破、缩量回调、放量下跌、高位放量滞涨等量价形态。
- 支持多种轻量机器学习算法：
  - KNN 相似样本。
  - 加权 KNN。
  - 逻辑回归。
- 输出历史相似上涨占比，用作买入评分参考。
- Web 单股分析展示 90 个交易日价格结构图，标注缠论分型点、笔和最近中枢。
- 输出缠论结构状态、当前位置、买点候选、风险结构和结构辅助建议，并将结构调整分纳入最终评分。
- 接入强势股回档策略：年线 MA250、放量突破年线、缩量回踩年线、年线向上拐头、新股放量突破首日高点会参与综合评分。
- 支持输入持仓成本价，输出单股持仓卖出判断、卖出风险评分、止盈/止损/减仓/继续持有建议。
- 输出买入、观察、持有或回避建议。
- 输出持仓卖出建议，包括继续持有、减仓观察、止盈/分批减仓、止损或仅技术风险提示。
- 输出买入依据、卖出依据、风险提示、关键指标和后续观察点。
- 提供 CLI 和本地 Web 前端。
- Web 前端支持抓取全 A 股票列表，边扫描边动态显示当前 Top 10，整轮完成后停止。
- CLI 支持定时扫描自定义股票池。

## 输出内容

一次分析会输出：

- 股票代码。
- 股票名称。
- 分析日期。
- 数据截止日。
- 最近 21 个交易日基础指标样本说明。
- 适合买入评分，其中机器学习概率分会叠加缠论结构调整分和年线/强势回档规则分。
- 适合卖出评分，即持仓卖出风险评分。
- 历史相似上涨占比。
- 买入依据和回避理由。
- 卖出依据，包括止盈、止损、减仓或继续持有理由。
- 风险提示。
- 关键量价指标。
- 缠论结构辅助，包括分型、笔、中枢、买点候选、风险结构和结构建议。
- 年线和强势回档信号，包括是否站上年线、年线是否向上、是否缩量回踩年线。
- 持仓卖出判断，包括持仓收益率、卖出风险评分、关键支撑位和卖出风险说明。
- 后续观察点。

## 本地运行

未安装为命令时，可以这样运行：

```bash
PYTHONPATH=src python3 -m cli advise 600519
```

指定机器学习算法：

```bash
PYTHONPATH=src python3 -m cli advise 600519 --algorithm weighted_knn
```

输出 JSON：

```bash
PYTHONPATH=src python3 -m cli advise 600519 --format json
```

分析单只股票的持仓卖出风险：

```bash
PYTHONPATH=src python3 -m cli sell 601988 --cost-price 5.20
```

卖出判断支持可选参数：

```bash
PYTHONPATH=src python3 -m cli sell 601988 --cost-price 5.20 --quantity 1000 --max-loss-rate 0.08 --target-profit-rate 0.20 --format json
```

CLI 扫描自定义股票池并输出 Top 10：

```bash
PYTHONPATH=src python3 -m cli scan --pool-file config/stock_pool.example.txt
```

只扫描一次：

```bash
PYTHONPATH=src python3 -m cli scan --pool-file config/stock_pool.example.txt --once
```

直接传入股票代码：

```bash
PYTHONPATH=src python3 -m cli scan --symbols 601988,600519,600036 --once
```

默认扫描间隔是 600 秒，也就是 10 分钟。可以调整：

```bash
PYTHONPATH=src python3 -m cli scan --pool-file config/stock_pool.example.txt --interval 600 --top 10
```

默认只推荐适合买入评分不低于 52 的股票，也就是至少达到 `WATCH` 级别。可以调整最低评分：

```bash
PYTHONPATH=src python3 -m cli scan --pool-file config/stock_pool.example.txt --min-score 60 --once
```

说明：CLI 股票池扫描保留最低适合买入评分过滤，适合小范围自选股筛选；Web 全 A 扫描不设置最低评分，会从已完成分析的股票中持续保留适合买入评分最高的 10 只。

## Web 前端

启动前端界面：

```bash
PYTHONPATH=src python3 -m web --port 8000
```

如果 `8000` 已被占用，服务会自动尝试 `8001` 到 `8010`。也可以让系统自动选择空闲端口：

```bash
PYTHONPATH=src python3 -m web --port 0
```

浏览器打开：

```text
http://127.0.0.1:8000
```

前端支持：

- 输入股票代码。
- 可选输入持仓成本价、持仓数量、最大亏损比例和目标止盈比例。
- 选择机器学习算法。
- 查看价格结构图、预测概率、缠论结构辅助、买入依据和卖出依据。
- 查看持仓卖出判断，包括卖出建议、卖出风险评分、持仓收益率和关键位。
- 抓取公开行情接口对应的全 A 股票列表。
- Web 全 A 扫描会剔除创业板、科创板和北交所股票，也就是 `300`、`301`、`688`、`689`、`4`、`8`、`9` 开头代码。
- 每轮全 A 扫描完成后停止，下一轮筛选需要手动重新启动。
- 动态显示扫描进度、当前入榜数量和 Top 10 买入观察池。
- Top 10 表格会展示缠论结构摘要，便于区分买点候选和结构风险。
- 无需用户登录。

全 A 扫描说明：

- 点击“全 A 扫描”后，选择算法，再点击“启动全 A 扫描”。
- 扫描运行中切换机器学习算法后，再点击“启动全 A 扫描”会重启扫描并使用新算法。
- 系统会逐页抓取全 A 股票列表，第一页股票拿到后就立刻开始逐只抓取日线数据并分析。
- 后台每分析完一只股票就更新候选结果，并通过事件流实时推送给前端，页面会立刻刷新已扫描数量、总数量和当前 Top 10。
- 必须等一整轮全 A 股票全部分析完成后，本轮 Top 10 才是完整结果；后台不会自动开始下一轮。
- 全 A 扫描请求数量较大，请根据网络情况耐心等待。

## 环境变量

可选：

```text
PUBLIC_DATA_USER_AGENT=
PUBLIC_DATA_TIMEOUT_SECONDS=
PUBLIC_STOCK_LIST_PAGE_SIZE=
PUBLIC_KLINE_DAYS=
MARKET_SCAN_WORKERS=
MARKET_TASK_TIMEOUT_SECONDS=
```

说明：

- `PUBLIC_DATA_USER_AGENT`：公开行情请求使用的浏览器标识。
- `PUBLIC_DATA_TIMEOUT_SECONDS`：公开行情请求超时时间，默认 `8` 秒。
- `PUBLIC_STOCK_LIST_PAGE_SIZE`：公开行情股票列表每页数量，默认 `80`。
- `PUBLIC_KLINE_DAYS`：公开日线接口拉取的历史交易日数量，默认 `260`，用于覆盖年线 MA250 和前一日年线。
- `MARKET_SCAN_WORKERS`：全 A 扫描并发线程数，默认 `4`。
- `MARKET_TASK_TIMEOUT_SECONDS`：单只股票分析任务最大等待时间，默认 `30` 秒，超时后会跳过该股票继续推进进度。

## 全 A 扫描

Web 全 A 扫描模块适合用来持续检查市场机会，找出适合买入评分最高的 Top 10。

扫描流程：

- 从新浪市场中心公开接口抓取全 A 股票代码和名称。
- 使用新浪 K 线公开接口抓取每只股票最新日线数据。
- 跳过创业板、科创板和北交所股票，`300`、`301`、`688`、`689`、`4`、`8`、`9` 开头代码不会进入分析队列。
- 每拿到一只股票，就提交日线数据抓取和量价分析任务。
- 使用最近 21 个交易日量价指标、机器学习模型和最近 90 个交易日缠论结构生成建议。
- 按适合买入评分和历史相似上涨占比排序，只保留当前 Top 10。
- 前端使用 `/api/market-scan/stream` 事件流接收实时更新；如果浏览器不支持事件流，只查询一次当前状态。
- 前端刷新状态文本直接展示后端返回的 `status_message`，不在前端自行拼接“本轮全量完成”等状态。
- 后台全量跑完一轮后停止，不会定时重新抓取或自动开始下一轮 Top 10 筛选。

相关接口：

- `POST /api/market-scan/start`：启动全 A 扫描。
- `POST /api/market-scan/stop`：停止全 A 扫描。
- `GET /api/market-scan/stream`：实时接收每只股票分析后的最新扫描状态。
- `GET /api/market-scan/status`：查询当前扫描状态，用作兜底和调试。

状态字段说明：

- `status`：机器可读的扫描状态，例如 `idle`、`loading_universe`、`scanning`、`finished`、`stopping`、`stopped`、`error`。
- `status_message`：后端生成的中文状态说明，前端直接展示这个字段。
- `fetched_count`：从股票列表中已经读取到的股票数量。
- `skipped_count`：因为创业板、科创板等过滤规则被跳过的股票数量。
- `failed_count`：数据抓取失败、数据不足或超时跳过的股票数量。
- `scanned_count`：已经完成分析的股票数量。
- `total_count`：当前已抓取到并进入扫描队列的股票数量。
- `top_results`：当前 Top 10 推荐列表。

扫描结果包括：

- 排名。
- 股票代码。
- 股票名称。
- 建议。
- 适合买入评分。
- 历史相似上涨占比。
- 缠论结构摘要。
- 核心理由。
- 主要风险。

扫描刚启动且还没有股票完成分析时，前端会显示等待提示；只要有股票完成分析，就会立刻参与 Top 10 排序。如果表格迟迟为空，可以先看 `status_message` 里的已跳过和失败数量，判断是板块过滤较多、行情请求失败，还是请求超时。

注意：全 A 扫描会产生大量行情请求。建议先使用默认并发数，避免把 `MARKET_SCAN_WORKERS` 设置过高。

如果扫描进度停在类似 `99/102`，通常表示最后几只股票的行情请求很慢或无响应。系统会按 `MARKET_TASK_TIMEOUT_SECONDS` 跳过超时股票，避免整轮扫描被少数请求卡住。

## CLI 股票池扫描

CLI 扫描模块适合定时检查一批关注股票，找出适合买入评分最高的 Top 10。

股票池文件格式：

```text
601988
600519
600036
```

说明：

- 每行一个 6 位股票代码。
- 空行会被忽略。
- 以 `#` 开头的行会被当作注释。
- 示例文件在 `config/stock_pool.example.txt`。

扫描结果会按适合买入评分和历史相似上涨占比排序，输出：

- 排名。
- 股票代码。
- 股票名称。
- 建议。
- 适合买入评分。
- 历史相似上涨占比。
- 缠论结构摘要。
- 核心理由。
- 主要风险。

如果股票池里没有达到最低评分的股票，系统会输出“暂无可推荐股票”。

注意：股票池越大，请求行情接口的次数越多。CLI 更适合自选股或重点观察池；全市场扫描请使用 Web 前端的全 A 扫描模块。

## 指标和评分

系统当前不是只看价格和成交量，而是把公开日线数据拆成多类指标：

- 基础行情：开盘价、最高价、最低价、收盘价、成交量、成交额。
- 涨跌幅：1 日、5 日、20 日涨跌幅。
- 均线：MA5、MA10、MA20、MA250 年线。
- 量能：5 日均量、20 日均量、成交量相对均量倍率。
- 流动性：20 日平均成交额。
- 突破和回调：20 日高点突破、年线突破、缩量回踩年线。
- 风险形态：放量下跌、放量跌破 20 日均线、高位放量滞涨、20 日涨幅过高。
- 机器学习：KNN、加权 KNN、逻辑回归输出历史相似上涨占比。
- 缠论结构：分型、笔、中枢、买点候选、顶背驰候选、跌破中枢候选。
- 持仓卖出：持仓收益率、最大亏损比例、目标止盈比例、盈利回撤、关键支撑位。

指标明细见 [股票指标总览](doc/stock_indicators.md)。

## 机器学习算法

当前支持三种算法：

| 算法 | 说明 |
| --- | --- |
| KNN 相似样本 | 查找历史上与当前量价状态最相似的样本，用相似样本的次日上涨比例作为买入概率；最近邻数量会按训练样本数动态调整。 |
| 加权 KNN | 与 KNN 类似，但越相似的历史样本权重越高；最近邻数量同样会按训练样本数动态调整。 |
| 逻辑回归 | 使用历史量价特征拟合次日上涨概率。 |

KNN 和加权 KNN 的最近邻数量按训练样本数动态选择：先取训练样本数平方根，再限制在 `7` 到 `21` 之间，并使用奇数以降低平票概率。默认拉取 `260` 个日线数据时，通常会生成约 `239` 个训练样本，最近邻数量约为 `15`。

适合买入评分会在机器学习概率分基础上叠加缠论结构调整分和年线/强势回档规则分。示例：

```text
适合买入评分 = 机器学习概率分 + 缠论结构调整分 + 年线/强势回档规则分
```

当前缠论调整分为第一版日线简化规则，常见含义包括：

- 三买候选、二买候选、上升结构会增加观察分。
- 顶背驰候选、跌破中枢候选、下降结构延续会降低评分。
- 调整后的分数会限制在 `0` 到 `100` 之间。

当前模型使用的主要特征包括：

- 1 日涨跌幅。
- 5 日涨跌幅。
- 20 日涨跌幅。
- 收盘价相对 5 日均线位置。
- 收盘价相对 20 日均线位置。
- 20 日均线斜率。
- 成交量 / 5 日均量。
- 成交量 / 20 日均量。
- 收盘价相对过去 20 日高点位置。

## 缠论结构辅助

系统当前实现的是第一版程序化缠论，只使用日线数据，不做分钟级、多级别递归或自动交易。

单股分析会基于最近 90 个交易日识别：

- 顶分型和底分型：图中标记为“顶”和“底”，用于表示局部转折点。
- 笔：连接相邻分型，区分简化上涨笔和下跌笔。
- 最近中枢：用价格重叠区间表示多空反复争夺的位置。
- 结构状态：上升结构、下降结构、盘整结构或不确定。
- 当前位置：中枢内震荡、中枢上沿附近、离开中枢上方、跌破中枢下方等。
- 买点候选：二买候选、三买候选或无。
- 风险结构：顶背驰候选、跌破中枢候选、下降结构延续等。

缠论结构会生成一个结构调整分，并进入适合买入评分：

```text
适合买入评分 = 机器学习概率分 + 缠论结构调整分 + 年线/强势回档规则分
```

典型加分项：

- 出现二买候选或三买候选。
- 处在上升结构中。
- 突破或回踩中枢后结构没有明显破坏。

典型扣分项：

- 跌破最近中枢下沿。
- 出现顶背驰候选。
- 下降结构延续。
- 高位离开中枢后缩量乏力。

缠论结构建议会出现在：

- Web 单股分析的“缠论结构辅助”板块。
- 价格结构图中的分型点、笔和中枢标注。
- 数据证据和风险提示。
- CLI 文本输出的“缠论结构辅助”区块。
- CLI / Web 扫描结果的缠论结构摘要。

注意：缠论结构只是辅助判断“当前位置是否适合观察”，不是买卖保证。所有结构信号都需要结合成交量、机器学习概率、市场环境和仓位控制使用。

## 注意事项

- 机器学习预测是统计判断，不是收益承诺。
- 当前系统只使用公开日线行情及其派生技术指标，不分析基本面、新闻、政策、资金流、盘口明细等信息。
- 输出建议适合作为辅助参考，不应作为唯一交易依据。
- 新手用户尤其要控制仓位，避免单票重仓。
- 数据来自公开行情接口，请遵守相关服务条款和法律法规。
