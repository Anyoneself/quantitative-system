# Quantitative-System

Quantitative-System 是一个面向 A 股单股分析和全市场扫描的量化辅助决策系统。

项目当前聚焦两个核心指标：**股票价格**和**成交量**。用户输入股票代码和机器学习算法后，系统会实时抓取该股票的最新日线数据，基于最近 21 个交易日计算量价指标，并用历史量价样本训练轻量机器学习模型，预测下一个交易日是否推荐买进。系统统一使用免登录公开行情接口。

这个项目不是自动交易系统，也不承诺盈利。它的定位是一个“股票量价分析助手”：帮助用户更清晰地看到当前股票的趋势、量能、风险、机器学习预测结果，以及全 A 市场里相对更值得观察的候选股票。

## 项目目标

- 输入一个股票代码，输出是否建议下一个交易日买进。
- 前端支持抓取全 A 股票列表，扫描过程中实时更新当前 Top 10，整轮完成后等待 10 分钟再进入下一轮。
- 给出机器学习预测概率，而不是只给一句模糊结论。
- 给出适合新手理解的详细理由、风险提示和后续观察点。
- 每次分析都实时抓取最新行情数据，不依赖本地缓存。
- 保持实现简单，避免冗余设计。

## 适用场景

- 想快速查看一只股票当前是否值得买入观察。
- 想了解一只股票最近 21 个交易日的量价状态。
- 想用简单机器学习模型辅助判断下一个交易日上涨概率。
- 新手用户想看到更容易理解的买入理由和风险解释。

## 当前功能

- 每次从公开行情接口抓取最新日线数据。
- 自动识别股票名称，例如 `601988` 对应 `中国银行`。
- 计算最近 21 个交易日的价格和成交量指标。
- 识别涨停缩量、放量突破、缩量回调、放量下跌等量价形态。
- 支持多种轻量机器学习算法：
  - KNN 相似样本。
  - 加权 KNN。
  - 逻辑回归。
- 输出历史相似上涨占比，用作下一个交易日买入参考。
- 输出买入、观察、持有或回避建议。
- 输出核心理由、风险提示、关键指标和新手解读。
- 提供 CLI 和本地 Web 前端。
- Web 前端支持抓取全 A 股票列表，边扫描边动态显示当前 Top 10，整轮完成后等待 10 分钟再重跑。
- CLI 支持定时扫描自定义股票池。

## 输出内容

一次分析会输出：

- 股票代码。
- 股票名称。
- 分析日期。
- 数据截止日。
- 最近 21 个交易日样本说明。
- 机器学习评分。
- 历史相似上涨占比。
- 核心买入或回避理由。
- 风险提示。
- 关键量价指标。
- 新手友好的后续观察点。

## 本地运行

未安装为命令时，可以这样运行：

```bash
PYTHONPATH=src python3 -m quant_system.cli advise 600519
```

指定机器学习算法：

```bash
PYTHONPATH=src python3 -m quant_system.cli advise 600519 --algorithm weighted_knn
```

输出 JSON：

```bash
PYTHONPATH=src python3 -m quant_system.cli advise 600519 --format json
```

CLI 扫描自定义股票池并输出 Top 10：

```bash
PYTHONPATH=src python3 -m quant_system.cli scan --pool-file config/stock_pool.example.txt
```

只扫描一次：

```bash
PYTHONPATH=src python3 -m quant_system.cli scan --pool-file config/stock_pool.example.txt --once
```

直接传入股票代码：

```bash
PYTHONPATH=src python3 -m quant_system.cli scan --symbols 601988,600519,600036 --once
```

默认扫描间隔是 600 秒，也就是 10 分钟。可以调整：

```bash
PYTHONPATH=src python3 -m quant_system.cli scan --pool-file config/stock_pool.example.txt --interval 600 --top 10
```

默认只推荐机器学习评分不低于 52 的股票，也就是至少达到 `WATCH` 级别。可以调整最低评分：

```bash
PYTHONPATH=src python3 -m quant_system.cli scan --pool-file config/stock_pool.example.txt --min-score 60 --once
```

说明：CLI 股票池扫描保留最低评分过滤，适合小范围自选股筛选；Web 全 A 扫描不设置最低评分，会从已完成分析的股票中持续保留评分最高的 10 只。

## Web 前端

启动前端界面：

```bash
PYTHONPATH=src python3 -m quant_system.web --port 8000
```

如果 `8000` 已被占用，服务会自动尝试 `8001` 到 `8010`。也可以让系统自动选择空闲端口：

```bash
PYTHONPATH=src python3 -m quant_system.web --port 0
```

浏览器打开：

```text
http://127.0.0.1:8000
```

前端支持：

- 输入股票代码。
- 选择机器学习算法。
- 查看图表、预测概率、建议和新手解读。
- 抓取公开行情接口对应的全 A 股票列表。
- Web 全 A 扫描会剔除创业板、科创板和北交所股票，也就是 `300`、`301`、`688`、`689`、`4`、`8`、`9` 开头代码。
- 每轮全 A 扫描完成后，等待 10 分钟再进行下一轮筛选。
- 动态显示扫描进度、当前入榜数量和 Top 10 买入观察池。
- 无需用户登录。

全 A 扫描说明：

- 点击“全 A 扫描”后，选择算法，再点击“启动全 A 扫描”。
- 扫描运行中切换机器学习算法后，再点击“启动全 A 扫描”会重启扫描并使用新算法。
- 系统会逐页抓取全 A 股票列表，第一页股票拿到后就立刻开始逐只抓取日线数据并分析。
- 后台每分析完一只股票就更新候选结果，并通过事件流实时推送给前端，页面会立刻刷新已扫描数量、总数量和当前 Top 10。
- 必须等一整轮全 A 股票全部分析完成后，后台才会等待 600 秒，也就是 10 分钟，再自动开始下一轮 Top 10 筛选。
- 全 A 扫描请求数量较大，请根据网络情况耐心等待。

## 环境变量

可选：

```text
PUBLIC_DATA_USER_AGENT=
PUBLIC_DATA_TIMEOUT_SECONDS=
PUBLIC_STOCK_LIST_PAGE_SIZE=
PUBLIC_KLINE_DAYS=
MARKET_SCAN_WORKERS=
MARKET_TASK_TIMEOUT_SECONDS=
```

说明：

- `PUBLIC_DATA_USER_AGENT`：公开行情请求使用的浏览器标识。
- `PUBLIC_DATA_TIMEOUT_SECONDS`：公开行情请求超时时间，默认 `8` 秒。
- `PUBLIC_STOCK_LIST_PAGE_SIZE`：公开行情股票列表每页数量，默认 `80`。
- `PUBLIC_KLINE_DAYS`：公开日线接口拉取的历史交易日数量，默认 `80`。
- `MARKET_SCAN_WORKERS`：全 A 扫描并发线程数，默认 `4`。
- `MARKET_TASK_TIMEOUT_SECONDS`：单只股票分析任务最大等待时间，默认 `30` 秒，超时后会跳过该股票继续推进进度。

## 全 A 扫描

Web 全 A 扫描模块适合用来持续检查市场机会，找出机器学习评分最高的 Top 10。

扫描流程：

- 从新浪市场中心公开接口抓取全 A 股票代码和名称。
- 使用新浪 K 线公开接口抓取每只股票最新日线数据。
- 跳过创业板、科创板和北交所股票，`300`、`301`、`688`、`689`、`4`、`8`、`9` 开头代码不会进入分析队列。
- 每拿到一只股票，就提交日线数据抓取和量价分析任务。
- 使用最近 21 个交易日量价指标和机器学习模型生成建议。
- 按机器学习评分和历史相似上涨占比排序，只保留当前 Top 10。
- 前端优先使用 `/api/market-scan/stream` 事件流接收实时更新；如果浏览器不支持事件流，再退回每 2 秒查询一次状态。
- 前端刷新状态文本直接展示后端返回的 `status_message`，不在前端自行拼接“本轮全量完成”等状态。
- 后台全量跑完一轮后，等待 10 分钟再开始下一轮扫描和 Top 10 筛选。

相关接口：

- `POST /api/market-scan/start`：启动全 A 扫描。
- `POST /api/market-scan/stop`：停止全 A 扫描。
- `GET /api/market-scan/stream`：实时接收每只股票分析后的最新扫描状态。
- `GET /api/market-scan/status`：查询当前扫描状态，用作兜底和调试。

状态字段说明：

- `status`：机器可读的扫描状态，例如 `idle`、`scanning`、`finished`、`waiting`。
- `status_message`：后端生成的中文状态说明，前端直接展示这个字段。
- `fetched_count`：从股票列表中已经读取到的股票数量。
- `skipped_count`：因为创业板、科创板等过滤规则被跳过的股票数量。
- `failed_count`：数据抓取失败、数据不足或超时跳过的股票数量。
- `scanned_count`：已经完成分析的股票数量。
- `total_count`：当前已抓取到并进入扫描队列的股票数量。
- `top_results`：当前 Top 10 推荐列表。

扫描结果包括：

- 排名。
- 股票代码。
- 股票名称。
- 建议。
- 机器学习评分。
- 历史相似上涨占比。
- 核心理由。
- 主要风险。

扫描刚启动且还没有股票完成分析时，前端会显示等待提示；只要有股票完成分析，就会立刻参与 Top 10 排序。如果表格迟迟为空，可以先看 `status_message` 里的已跳过和失败数量，判断是板块过滤较多、行情请求失败，还是请求超时。

注意：全 A 扫描会产生大量行情请求。建议先使用默认并发数，避免把 `MARKET_SCAN_WORKERS` 设置过高。

如果扫描进度停在类似 `99/102`，通常表示最后几只股票的行情请求很慢或无响应。系统会按 `MARKET_TASK_TIMEOUT_SECONDS` 跳过超时股票，避免整轮扫描被少数请求卡住。

## CLI 股票池扫描

CLI 扫描模块适合定时检查一批关注股票，找出机器学习评分最高的 Top 10。

股票池文件格式：

```text
601988
600519
600036
```

说明：

- 每行一个 6 位股票代码。
- 空行会被忽略。
- 以 `#` 开头的行会被当作注释。
- 示例文件在 `config/stock_pool.example.txt`。

扫描结果会按机器学习评分和历史相似上涨占比排序，输出：

- 排名。
- 股票代码。
- 股票名称。
- 建议。
- 机器学习评分。
- 历史相似上涨占比。
- 核心理由。
- 主要风险。

如果股票池里没有达到最低评分的股票，系统会输出“暂无可推荐股票”。

注意：股票池越大，请求行情接口的次数越多。CLI 更适合自选股或重点观察池；全市场扫描请使用 Web 前端的全 A 扫描模块。

## 机器学习算法

当前支持三种算法：

| 算法 | 说明 |
| --- | --- |
| KNN 相似样本 | 查找历史上与当前量价状态最相似的样本，用相似样本的次日上涨比例作为买入概率。 |
| 加权 KNN | 与 KNN 类似，但越相似的历史样本权重越高。 |
| 逻辑回归 | 使用历史量价特征拟合次日上涨概率。 |

当前模型使用的主要特征包括：

- 1 日涨跌幅。
- 5 日涨跌幅。
- 20 日涨跌幅。
- 收盘价相对 5 日均线位置。
- 收盘价相对 20 日均线位置。
- 20 日均线斜率。
- 成交量 / 5 日均量。
- 成交量 / 20 日均量。
- 收盘价相对过去 20 日高点位置。

## 注意事项

- 机器学习预测是统计判断，不是收益承诺。
- 当前系统只分析价格和成交量，不分析基本面、新闻、政策、资金流等信息。
- 输出建议适合作为辅助参考，不应作为唯一交易依据。
- 新手用户尤其要控制仓位，避免单票重仓。
- 数据来自公开行情接口，请遵守相关服务条款和法律法规。
