# 缠论在系统中的使用设计

如果你还不了解缠论，建议先阅读 [缠论小白说明：它是什么，如何帮助股票决策](chan_theory_for_beginners.md)。本文档更偏系统实现设计。

## 1. 定位

缠论在本系统中不作为单独的买卖决策引擎，而作为价格结构分析层，辅助当前的量价模型判断股票是否处在更值得观察的位置。

当前系统已经可以基于价格、成交量和机器学习相似样本筛出全 A Top 10。但这些结果主要回答的是：

```text
当前量价状态和历史相似样本相比，下一交易日上涨占比如何？
```

缠论层要补充回答的是：

```text
当前股票处在什么结构位置？
上涨是离开中枢、回踩确认，还是高位背驰后的风险释放？
```

因此，缠论更适合用于：

- 过滤掉看似高分但结构位置不好的股票。
- 标记更接近买点或观察点的股票。
- 解释当前价格所处的趋势、盘整和背驰状态。
- 为后续回测提供可验证的结构信号。

缠论信号不能直接等同于盈利概率。所有买点、卖点和背驰判断都必须经过回测验证。

## 2. 使用边界

### 2.1 当前阶段只使用日线数据

系统当前只抓取日线行情，因此缠论分析先限制在日线级别。

日线级别可以做：

- K 线包含关系处理。
- 顶分型和底分型识别。
- 笔的初步划分。
- 线段和中枢的简化识别。
- 日线级别的一买、二买、三买候选标记。
- 日线级别背驰和盘整背驰候选标记。

日线级别暂不做：

- 分钟级别精确买卖点。
- 多级别联立确认。
- 盘中实时中枢变化。
- 精细到成交明细或盘口的背驰判断。

### 2.2 缠论只输出结构信号

缠论模块只输出结构化信号，不直接输出“买入”或“卖出”。

建议输出：

```text
结构趋势：上升 / 下降 / 盘整 / 不确定
当前位置：中枢内 / 中枢上沿 / 中枢下沿 / 离开中枢 / 回踩中枢
买点候选：一买候选 / 二买候选 / 三买候选 / 无
风险候选：顶背驰候选 / 跌破中枢候选 / 结构破坏候选 / 无
置信度：0-100
解释文本：为什么识别为该结构
```

最终是否进入 Top 10，仍由综合评分决定。

## 3. 缠论概念到系统字段的映射

### 3.1 K 线包含处理

缠论的分型和笔识别需要先处理包含关系。

系统输入是 `PriceVolumeBar`：

```text
trade_date
open
high
low
close
volume
amount
```

包含处理主要使用：

- `high`
- `low`
- 前后 K 线方向

处理目标是得到没有包含关系的简化 K 线序列，减少分型误判。

### 3.2 分型

顶分型：

```text
中间 K 线的 high 高于左右两侧 high
且中间 K 线的 low 高于左右两侧 low
```

底分型：

```text
中间 K 线的 low 低于左右两侧 low
且中间 K 线的 high 低于左右两侧 high
```

系统中可表示为：

```python
ChanFractal(
    kind="top" | "bottom",
    index=int,
    trade_date=date,
    high=float,
    low=float,
)
```

### 3.3 笔

笔用于描述一段明确的上升或下降结构。

简化规则：

- 顶分型和底分型交替出现。
- 顶底分型之间至少间隔一定数量 K 线。
- 上升笔从底分型连接到后续顶分型。
- 下降笔从顶分型连接到后续底分型。

系统中可表示为：

```python
ChanStroke(
    direction="up" | "down",
    start_index=int,
    end_index=int,
    start_price=float,
    end_price=float,
)
```

### 3.4 线段

线段由多笔组成，用来刻画更稳定的结构方向。

当前阶段可以先使用简化版本：

- 至少由 3 笔构成。
- 上升线段高低点整体抬高。
- 下降线段高低点整体降低。

线段不必一开始实现复杂破坏规则，先服务于趋势方向和中枢识别。

### 3.5 中枢

中枢代表一段价格重叠区域，是缠论中判断趋势延续和盘整的重要结构。

简化规则：

- 至少 3 段相邻走势存在价格区间重叠。
- 中枢上沿为重叠区间高点。
- 中枢下沿为重叠区间低点。

系统中可表示为：

```python
ChanCenter(
    start_index=int,
    end_index=int,
    upper=float,
    lower=float,
    middle=float,
)
```

### 3.6 背驰

背驰用于识别趋势力度减弱。

当前系统没有 MACD 等复杂指标，也没有分钟级数据，因此初期使用量价力度近似判断。

上涨背驰候选：

```text
价格创新高
但 5 日涨幅、20 日涨幅、成交量放大程度或离开中枢力度弱于前一段上涨
```

下跌背驰候选：

```text
价格创新低
但下跌幅度、成交量放大程度或离开中枢力度弱于前一段下跌
```

背驰候选只作为风险提示或观察提示，不直接作为确定买卖点。

## 4. 与现有量价模型的关系

当前模型会输出：

- 机器学习评分。
- 历史相似上涨占比。
- KNN 最近邻样本数。
- 价格趋势证据。
- 成交量证据。
- 风险提示。

缠论层加入后，建议形成三层判断：

```text
第一层：流动性和基础过滤
第二层：机器学习量价评分
第三层：缠论结构过滤和解释
```

### 4.1 对 Top 10 的影响

不建议缠论直接替代原评分。建议先作为加减分项：

加分：

- 二买候选。
- 三买候选。
- 离开中枢后回踩不破。
- 结构趋势向上且未出现顶背驰候选。
- 中枢上移，且成交量健康配合。

减分：

- 高位顶背驰候选。
- 跌回中枢且放量下跌。
- 跌破最近中枢下沿。
- 结构仍处在下降线段中。
- 量价模型高分但缠论结构处在高位离开段末端。

### 4.2 建议评分方式

初期不需要复杂模型，可以增加一个 `chan_score_adjustment`：

```text
三买候选：+8
二买候选：+6
离开中枢后缩量回踩不破：+5
上升结构中：+4
盘整结构中：0
下降结构中：-8
顶背驰候选：-10
跌破中枢下沿：-12
结构无法识别：0
```

最终评分：

```text
综合评分 = 机器学习评分 + 缠论结构调整分
```

综合评分仍需限制在 `0` 到 `100` 之间。

## 5. 买点候选定义

### 5.1 一买候选

一买风险较高，初期不建议作为 Top 10 主要来源。

可识别条件：

- 前面存在明显下降结构。
- 下跌创新低后力度减弱。
- 出现底分型。
- 放量下跌没有继续扩大，或后续上涨有成交量确认。

系统用途：

- 标记为“激进观察”。
- 不直接加高分。
- 需要后续二买或趋势修复确认。

### 5.2 二买候选

二买更适合当前系统作为观察信号。

可识别条件：

- 前面出现疑似一买后的反弹。
- 回调不创新低。
- 回调成交量缩小。
- 出现底分型或重新站上关键均线。

系统用途：

- 作为较高质量观察信号。
- 和 KNN 高相似上涨占比结合时加分。

### 5.3 三买候选

三买适合用来识别趋势延续。

可识别条件：

- 价格离开最近中枢上沿。
- 回踩不跌回中枢。
- 回踩时成交量缩小。
- 再次放量上行或保持强势。

系统用途：

- 作为趋势型 Top 10 的主要候选。
- 如果同时机器学习评分较高，可以提高排序。

## 6. 输出展示

前端和 CLI 可以增加“缠论结构”区块。

建议展示字段：

```text
结构状态：上升结构 / 盘整结构 / 下降结构 / 不确定
中枢区间：下沿 - 上沿
当前位置：中枢内 / 上沿附近 / 离开中枢 / 回踩中枢
买点候选：二买候选 / 三买候选 / 无
风险结构：顶背驰候选 / 跌破中枢 / 无
结构解释：最近一段走势如何形成该判断
```

示例输出：

```text
缠论结构：上升结构
中枢区间：10.20 - 10.86
当前位置：离开中枢后回踩不破
买点候选：三买候选
风险结构：未发现明显顶背驰候选
解释：价格突破中枢上沿后回踩未跌回中枢，回踩阶段成交量低于 20 日均量，结构偏正面。
```

## 7. 推荐实现路径

### 7.1 第一版：只做结构标记

新增模块：

```text
src/analysis/chan_theory.py
```

第一版函数：

```python
def analyze_chan_structure(bars: list[PriceVolumeBar]) -> ChanStructureResult:
    ...
```

第一版只做：

- 包含关系处理。
- 顶底分型识别。
- 笔识别。
- 最近中枢识别。
- 二买、三买、顶背驰候选标记。

不做：

- 多级别联立。
- 自动交易。
- 复杂递归线段。
- 没有回测依据的强买卖结论。

### 7.2 第二版：接入评分

在 `advisor.py` 中加入缠论结构证据：

```text
如果三买候选且无顶背驰候选，增加理由和评分。
如果顶背驰候选或跌破中枢下沿，增加风险并扣分。
```

在 `market_scanner.py` 中让 Top 10 按综合评分排序。

### 7.3 第三版：回测验证

回测问题必须明确：

```text
当日识别为二买 / 三买候选后，未来 1 日、5 日、20 日收益如何？
加入缠论过滤后，Top 10 胜率是否提升？
最大回撤是否下降？
错过了多少原本盈利的股票？
```

只有回测证明有效后，才把缠论分数作为正式排序因子。

## 8. 风险与注意事项

- 缠论结构识别存在主观性，程序化实现必须明确规则，不能依赖模糊解释。
- 日线级别信号滞后，不能用于盘中精确买卖。
- 背驰候选不是确定反转，只能作为风险提示。
- 二买、三买候选也可能失败，必须结合止损和仓位控制。
- 当前公开行情数据未处理复权、停牌、涨跌停细节时，结构识别可能失真。
- 缠论加入系统后，必须通过回测比较加入前后的胜率、收益和回撤变化。

## 9. 最小可验收标准

缠论第一版完成时，至少应满足：

- 给定一组小型日线样本，可以稳定识别顶分型和底分型。
- 可以输出最近一段笔方向。
- 可以识别一个简化中枢区间。
- 可以标记是否为二买候选或三买候选。
- 可以输出中文解释文本。
- 单元测试覆盖包含关系、分型、笔、中枢和买点候选。

只有满足以上条件，缠论才算真正进入系统，而不是只停留在概念描述。
