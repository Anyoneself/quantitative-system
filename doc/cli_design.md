# CLI 功能设计

## 1. CLI 定位

CLI 的第一优先功能是单股决策分析。

用户输入一个股票代码，系统每次从公开行情抓取最新价格和成交量数据，计算量价信号，并使用机器学习模型预测下一个交易日是否推荐买进，最终输出：

- 股票名称，例如 `601988` 对应 `中国银行`。
- 是否建议下一个交易日买入、观察、持有或回避。
- 对应理由。
- 风险提示。
- 关键量价指标。
- 后续观察点。

CLI 不直接承诺盈利，也不直接替用户下单。它的定位是“量价分析助手”和“交易决策参考工具”。

## 2. 核心命令

第一版只保留一个核心命令：

```bash
quant advise 600519
```

可选参数：

```bash
quant advise 600519 --format json
```

参数说明：

- `symbol`：股票代码，必填。
- `--format`：输出格式，支持 `text` 和 `json`。

CLI 每次都会从公开行情抓取数据，不读取本地缓存。

## 3. 环境变量

第一版不需要必需环境变量。可选环境变量：

```text
PUBLIC_DATA_USER_AGENT=
PUBLIC_DATA_TIMEOUT_SECONDS=
PUBLIC_KLINE_DAYS=
```

说明：

- `PUBLIC_DATA_USER_AGENT`：请求使用的浏览器标识，不配置时使用默认值。
- `PUBLIC_DATA_TIMEOUT_SECONDS`：公开行情请求超时时间，不配置时使用默认值。
- `PUBLIC_KLINE_DAYS`：公开日线接口拉取的历史交易日数量，不配置时使用默认值。

暂不需要券商 API、实盘账户、数据库密码或其他数据商 Token。第一版 CLI 只负责分析，不负责下单。

## 4. 输出建议类型

系统只输出有限的建议类型，避免模糊表达。

```text
BUY        建议买入
WATCH      建议观察
HOLD       建议持有
REDUCE     建议减仓
SELL       建议卖出
AVOID      建议回避
NO_DATA    数据不足
```

含义：

- `BUY`：量价结构偏强，风险可控，满足买入观察条件。
- `WATCH`：出现部分积极信号，但还缺少确认。
- `HOLD`：已经持有时可以继续观察，暂未触发卖出条件。
- `REDUCE`：保留给后续持仓分析，当前未持仓场景一般不输出。
- `SELL`：保留给后续持仓分析，当前未持仓场景一般不输出。
- `AVOID`：风险较高，不适合买入。
- `NO_DATA`：数据不足，无法判断。

## 5. 输出格式

建议使用人类可读的文本输出。

示例：

```text
股票代码：600519
股票名称：贵州茅台
分析日期：2026-06-24
数据截止日：2026-06-23
样本区间：最近 21 个交易日
建议：WATCH
机器学习评分：56

核心理由：
1. 股价位于 20 日均线上方，短期趋势偏强。
2. 今日成交量低于 5 日均量，卖压暂时不大。
3. 未出现放量突破，买入确认不足。

风险提示：
1. 近 20 日涨幅较高，追高风险上升。
2. 若次日放量跌破 20 日均线，应降低评分。

关键指标：
- 收盘价：xxx
- 1 日涨跌幅：x.xx%
- 5 日涨跌幅：x.xx%
- 近 20 交易日涨跌幅：x.xx%
- 成交量 / 5 日均量：x.xx
- 成交量 / 20 日均量：x.xx
- 是否涨停：否
- 是否放量突破：否
- 是否缩量回调：是

后续观察：
1. 是否放量突破近期高点。
2. 回调是否继续缩量。
3. 是否跌破 20 日均线。

机器学习预测：
- 下一个交易日买入成功概率：56.00%
- 训练样本数：5900
- 最近邻样本数：9
- 最近邻上涨样本数：5
```

后续可以增加 JSON 输出，方便程序调用：

```bash
quant advise 600519 --format json
```

## 6. 分析流程

```text
输入股票代码
  -> 校验股票代码格式
  -> 从公开行情抓取最新日线数据
  -> 检查数据完整性
  -> 计算价格指标
  -> 计算成交量指标
  -> 识别量价形态
  -> 使用历史量价样本训练机器学习模型
  -> 预测下一个交易日买入成功概率
  -> 生成建议和理由
  -> 输出理由和风险
```

## 7. 第一版量价指标

价格指标：

- 最新收盘价。
- 1 日涨跌幅。
- 5 日涨跌幅。
- 20 日涨跌幅。
- 5 日均线。
- 10 日均线。
- 20 日均线。
- 20 日新高。
- 20 日新低。
- 是否涨停。
- 是否跌停。

成交量指标：

- 最新成交量。
- 5 日平均成交量。
- 20 日平均成交量。
- 成交量 / 5 日均量。
- 成交量 / 20 日均量。
- 最新成交额。
- 20 日平均成交额。

量价形态：

- 涨停缩量。
- 放量突破。
- 缩量回调。
- 放量下跌。
- 高位放量滞涨。
- 价涨量缩。
- 价跌量增。

## 8. 机器学习评分规则

第一版使用轻量 KNN 机器学习模型预测下一个交易日买入成功概率，不额外引入第三方机器学习依赖。

训练方式：

- 每次从公开行情抓取该股票完整日线数据。
- 从历史数据中滚动生成量价特征。
- 每个历史样本使用当日之前最近 21 个交易日特征。
- 标签为下一个交易日收盘价是否高于当日收盘价。
- 对当前最新交易日生成特征，寻找历史上最相似的 K 个样本。
- 用相似样本中上涨样本占比作为下一个交易日买入成功概率。

模型特征：

- 1 日涨跌幅。
- 5 日涨跌幅。
- 20 日涨跌幅。
- 收盘价相对 5 日均线位置。
- 收盘价相对 20 日均线位置。
- 20 日均线斜率。
- 成交量 / 5 日均量。
- 成交量 / 20 日均量。
- 收盘价相对过去 20 日高点位置。

建议映射：

```text
机器学习评分 >= 60：BUY
52 <= 机器学习评分 < 60：WATCH
45 <= 机器学习评分 < 52：HOLD
机器学习评分 < 45：AVOID
数据不足：NO_DATA
```

机器学习评分等于下一个交易日买入成功概率乘以 100。

量价规则仍用于解释理由和风险提示，例如趋势向上、放量突破、缩量涨停、放量跌破均线等，但不再直接累加成最终评分。

## 9. 理由生成规则

每个建议必须带理由，理由来自明确的量价规则。

理由不允许空泛表达，例如：

```text
走势不错，可以考虑。
```

理由必须可追溯，例如：

```text
收盘价高于 20 日均线，且 20 日均线向上，短期趋势偏强。
```

风险提示也必须来自明确规则，例如：

```text
近 20 日涨幅超过阈值，继续追高的回撤风险上升。
```

## 10. 最小实现模块

第一版只需要以下模块：

```text
src/quant_system/
├── cli.py
├── data/
│   ├── models.py
│   └── public_sources.py
├── analysis/
│   ├── indicators.py
│   ├── ml_model.py
│   ├── patterns.py
│   └── advisor.py
└── output/
    └── formatter.py
```

模块职责：

- `cli.py`：解析命令行参数。
- `models.py`：定义价格和成交量数据结构。
- `public_sources.py`：每次从公开行情抓取最新日线数据。
- `indicators.py`：计算价格和成交量指标。
- `ml_model.py`：训练轻量机器学习模型并预测下一个交易日买入概率。
- `patterns.py`：识别涨停缩量、放量突破等形态。
- `advisor.py`：根据机器学习预测和量价解释生成建议。
- `formatter.py`：格式化输出文本或 JSON。

## 11. 后续增强

第一版稳定后，再考虑扩展：

- 支持用户输入持仓成本和持仓数量。
- 支持 `--position held` 区分持有和未持有。
- 支持输出最近 N 日量价表。
- 支持同行业或同题材对比。
- 支持市场环境过滤。
- 支持历史建议回放和胜率统计。
- 支持保存每次分析结果，便于复盘。
