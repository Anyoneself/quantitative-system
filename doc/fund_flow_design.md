# 资金流向接入设计

## 0. 当前实现状态

第一版资金流向已经接入系统：

- 数据模块：`src/data/fund_flow.py`
- 分析模块：`src/analysis/fund_flow.py`
- 买入接口：`POST /api/advise` 返回 `fund_flow`
- 卖出接口：`POST /api/sell-advice` 返回 `fund_flow_risk`，并在 `sell_advice.fund_flow` 中保留详情
- 前端：单股分析页展示“资金流向”板块
- CLI：`advise` 和 `sell` 文本/JSON 输出包含资金流信息

当前第一版已经支持：

- 东方财富单股日级资金流抓取。
- 当日、3 日、5 日、10 日主力净流入统计。
- 5 日 / 10 日净流入天数。
- 超大单净流入和小单净流出辅助判断。
- 资金流调整分进入适合买入评分。
- 资金流风险进入适合卖出评分。
- 资金流数据不可用时降级，不阻断原有分析。

仍待后续完善：

- 资金流接口缓存过期时间和失败重试策略。
- 更多数据源兜底。
- 资金流进入机器学习特征前的回测验证。
- 全 A 扫描中资金流请求量控制和限流策略。

## 1. 定位

当前系统主要使用：

```text
价格 + 成交量 + 均线 + 年线 + 量价形态 + 机器学习 + 缠论结构
```

这些信息可以回答：

```text
股票价格是否走强？成交量是否配合？结构位置是否适合观察？
```

但它们不能直接回答：

```text
资金是否真的在持续流入？
上涨是主动买盘推动，还是缩量反弹？
下跌是资金出逃，还是普通回调？
```

资金流向模块要补充的就是这一层判断。

资金流向不应该单独决定买卖，而应该作为：

```text
买入评分的确认因子
卖出评分的风险因子
全 A 扫描排序的增强因子
```

## 2. 使用边界

资金流向数据存在几个天然问题：

- 不同平台的资金流口径不完全一致。
- 主力资金、大单、中单、小单的划分标准可能不同。
- 免费接口可能有延迟、缺失或频率限制。
- 单日资金净流入容易被盘中拉抬或尾盘异动干扰。

因此，第一版只做辅助判断，不直接输出“买入”或“卖出”。

建议定位为：

```text
资金流向确认：提高或降低当前信号可信度
```

## 3. 推荐数据源

### 3.1 东方财富资金流接口

优先考虑东方财富公开资金流数据，因为字段相对完整，常见字段包括：

- 主力净流入。
- 超大单净流入。
- 大单净流入。
- 中单净流入。
- 小单净流入。
- 主力净流入占比。
- 资金流排名。

第一版可以先使用单股日级别资金流，不做分钟级资金流。

### 3.2 备选数据源

如果东方财富接口不稳定，可考虑：

- 新浪资金流。
- 腾讯行情资金流。
- 第三方公开聚合接口。

但所有资金流数据源都必须经过字段校验和异常处理，不能直接相信返回值。

## 6. 核心资金流指标

### 6.1 当日主力净流入

字段：

```text
latest_main_net_inflow
```

用途：

```text
判断当天主力资金是净流入还是净流出。
```

单日指标不能单独作为买卖依据，只作为短线资金状态参考。

### 6.2 主力净流入占比

字段：

```text
latest_main_net_inflow_ratio
```

公式：

```text
主力净流入占比 = 主力净流入 / 当日成交额
```

用途：

```text
避免大市值股票因绝对金额大而天然占优。
```

### 6.3 3 日 / 5 日 / 10 日主力净流入

字段：

```text
main_inflow_3d
main_inflow_5d
main_inflow_10d
```

用途：

```text
判断资金是否连续流入，而不是单日异动。
```

### 6.4 5 日 / 10 日净流入天数

字段：

```text
positive_days_5d
positive_days_10d
```

用途：

```text
判断资金流入的连续性和稳定性。
```

### 6.5 超大单净流入

字段：

```text
super_large_inflow_5d
```

用途：

```text
识别更偏机构或大资金行为的流入迹象。
```

### 6.6 小单净流出

字段：

```text
retail_outflow_5d
```

用途：

```text
如果主力流入、小单流出，可能说明筹码正在从散户向大资金集中。
```

## 7. 买入评分接入方式

资金流向建议作为买入评分的加减分项。

### 7.1 加分场景

```text
5 日主力净流入为正：+4
10 日主力净流入为正：+4
5 日内至少 3 天主力净流入：+4
主力净流入占比超过 3%：+5
超大单 5 日净流入为正：+4
小单 5 日净流出且主力净流入：+3
放量突破 + 主力净流入：+6
三买候选 + 主力净流入：+6
```

### 7.2 扣分场景

```text
5 日主力净流出：-5
10 日主力净流出：-5
5 日内至少 4 天主力净流出：-6
主力净流出占比低于 -3%：-6
放量下跌 + 主力净流出：-8
跌破中枢 + 主力净流出：-8
跌破年线 + 主力净流出：-10
```

### 7.3 综合买入评分

接入资金流后，买入评分建议变成：

```text
适合买入评分 = 机器学习概率分
             + 缠论结构调整分
             + 年线/强势回档规则分
             + 资金流向调整分
```

资金流向调整分建议限制在：

```text
-20 到 +20
```

避免资金流单因子过度主导最终判断。

## 8. 卖出评分接入方式

卖出模块中，资金流主要作为风险增强因子。

### 8.1 增加卖出风险

```text
持仓盈利 + 主力连续净流出：卖出风险 +10
顶背驰候选 + 主力净流出：卖出风险 +12
跌破中枢 + 主力净流出：卖出风险 +12
放量跌破 20 日均线 + 主力净流出：卖出风险 +15
跌破年线 + 主力净流出：卖出风险 +15
```

### 8.2 降低卖出风险

```text
缩量回调 + 主力净流入：卖出风险 -6
年线上方回踩 + 主力净流入：卖出风险 -6
缠论结构未破坏 + 主力净流入：卖出风险 -4
```

卖出风险分仍然应该优先保护本金，资金流不能覆盖硬止损信号。

## 9. 和现有模块的关系

### 9.1 与机器学习模型

资金流可以作为机器学习特征进入未来模型。

建议新增特征：

- 5 日主力净流入占成交额比例。
- 10 日主力净流入占成交额比例。
- 5 日净流入天数比例。
- 超大单净流入占比。
- 小单净流出占比。

但第一版建议先作为规则分接入，等有回测后再进入机器学习特征。

### 9.2 与缠论结构

资金流可以用来确认缠论买点或风险点。

示例：

```text
三买候选 + 主力净流入 -> 买点可信度提高
顶背驰候选 + 主力净流出 -> 止盈风险提高
跌破中枢 + 主力净流出 -> 趋势破坏风险提高
```

### 9.3 与强势回档策略

示例：

```text
缩量回踩年线 + 主力净流入 -> 强势回档可信度提高
跌破年线 + 主力净流出 -> 长期趋势破坏风险提高
```

## 13. 异常处理

如果资金流数据获取失败：

- 不阻断原有买入/卖出分析。
- 前端显示“资金流数据暂不可用”。
- 综合评分不加入资金流调整分。
- 日志或响应中保留错误原因，便于排查。

示例：

```text
资金流数据暂不可用，本次评分仅使用价格、成交量、缠论结构和机器学习结果。
```

## 14. 最小可验收标准

第一版完成时，至少应满足：

- 可以按股票代码拉取最近若干日资金流数据。
- 可以输出当日、3 日、5 日、10 日主力净流入。
- 可以识别持续流入、持续流出、资金分歧三类信号。
- 可以把资金流调整分加入买入评分。
- 可以把资金流风险加入卖出评分。
- 前端展示资金流向板块。
- CLI 能输出资金流说明。
- 单元测试覆盖资金持续流入、持续流出、数据缺失三种情况。

## 15. 风险说明

资金流数据不是绝对真实的资金行为，只是平台基于成交单划分得到的估算结果。

使用时必须注意：

- 资金流不能替代价格趋势。
- 单日主力净流入可能是假突破或尾盘拉抬。
- 主力净流出也可能是洗盘或调仓。
- 必须结合价格结构、成交量、缠论位置和回测结果使用。

资金流接入后，最重要的验证不是看个别案例，而是回测：

```text
加入资金流调整分后，Top 10 胜率是否提升？
平均收益是否提升？
最大回撤是否下降？
卖出风险评分是否更早识别风险？
```
