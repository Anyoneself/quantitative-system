# 单股卖出判断设计

## 0. 当前实现状态

第一版卖出判断已经接入系统：

- 后端模块：`src/analysis/sell_advisor.py`
- CLI 命令：`PYTHONPATH=src python3 -m cli sell 601988 --cost-price 5.20`
- Web 接口：`POST /api/sell-advice`
- Web 前端：单股分析表单支持输入持仓成本价、持仓数量、最大亏损比例和目标止盈比例，结果区展示“持仓卖出判断”。

当前第一版已经支持：

- 计算持仓收益率。
- 输出卖出风险评分 `sell_risk_score`。
- 输出 `继续持有 / 减仓观察 / 止盈 / 止损 / 仅技术风险提示`。
- 复用 20 日均线、年线、放量下跌、高位放量滞涨、缠论中枢和背驰候选等风险信号。

仍待后续完善：

- ATR 动态止损。
- 更精确的盈利回撤跟踪。
- 卖出规则回测和不同风格参数校准。

## 1. 定位

当前系统主要回答的是：

```text
这只股票现在是否适合买入观察？
```

卖出判断要补充回答的是：

```text
如果我已经持有这只股票，现在是继续拿、减仓、止盈，还是应该止损离场？
```

因此，卖出模块不应该复用“买入建议”的单一结论，而应该单独输出持仓视角的建议：

```text
继续持有 / 减仓观察 / 止盈 / 止损 / 不适用
```

买入判断关注的是“是否值得开仓”，卖出判断关注的是“持仓风险是否已经超过继续持有的价值”。这两个问题不能混在一个评分里。

## 2. 使用边界

卖出判断必须区分两类场景：

```text
未持有：只能给买入观察建议，不给卖出建议。
已持有：需要结合持仓成本、持仓收益、最大可承受亏损和趋势结构判断卖出。
```

第一版可以先支持用户输入：

```text
股票代码
持仓成本价
持仓数量，可选
最大可承受亏损比例，可选，默认 8%
目标止盈比例，可选，默认 20%
```

如果用户没有输入持仓成本，系统只能输出“技术风险提示”，不能直接判断是否止盈或止损。

## 3. 卖出判断的核心原则

### 3.1 不因为上涨就立刻卖

上涨本身不是卖出理由。

真正需要考虑卖出的情况是：

- 上涨后出现放量滞涨。
- 缠论结构出现顶背驰候选。
- 跌破关键支撑位。
- 盈利已经较高但趋势动能明显减弱。

### 3.2 不因为下跌就立刻卖

下跌也不一定意味着应该卖。

强势股在上涨趋势中经常会缩量回调。如果回调没有跌破关键结构位，反而可能是继续持有或观察的信号。

需要卖出的下跌通常具备这些特征：

- 放量下跌。
- 跌破 20 日均线。
- 跌破年线。
- 跌破缠论中枢下沿。
- 持仓亏损超过预设止损线。

### 3.3 卖出判断优先保护本金

卖出系统的第一目标不是卖在最高点，而是：

```text
避免小亏变大亏，避免盈利大幅回吐。
```

因此，止损和趋势破坏信号的优先级要高于继续持有信号。

## 4. 当前已有指标如何用于卖出

### 4.1 价格跌破 20 日均线

已有字段：

```text
close
ma20
close_above_ma20
```

卖出含义：

```text
短中期趋势转弱，适合进入减仓观察。
```

如果同时出现放量下跌，风险等级提高。

### 4.2 20 日均线向下

已有字段：

```text
ma20
ma20_prev
ma20_up
```

卖出含义：

```text
中期趋势不再向上，继续持有的胜率下降。
```

### 4.3 放量下跌

已有字段：

```text
price_down_volume_up
volume_drop_below_ma20
high_volume_drop
```

卖出含义：

```text
卖压明显增强，需要考虑减仓或止损。
```

### 4.4 高位放量滞涨

已有字段：

```text
high_volume_stagnation
high_20d_return
```

卖出含义：

```text
股票短期涨幅较大，但放量后价格无法继续上涨，可能出现筹码松动。
```

这类信号更适合止盈或减仓，而不是止损。

### 4.5 跌破年线

已有字段：

```text
ma250
close_above_ma250
```

卖出含义：

```text
长期趋势重新转弱，已经持有时应降低仓位或离场。
```

### 4.6 缠论结构风险

已有字段：

```text
chan_structure.risk_signal
chan_structure.position
chan_structure.center
```

卖出含义：

- 顶背驰候选：考虑止盈或减仓。
- 跌破中枢候选：考虑减仓或止损。
- 下降结构延续：不适合继续重仓持有。
- 高位缩量候选：考虑锁定部分利润。

## 5. 需要新增的卖出专用字段

### 5.1 持仓收益率

输入字段：

```text
cost_price
```

计算：

```text
holding_return = (close - cost_price) / cost_price
```

用途：

- 判断是否触发止损。
- 判断是否达到止盈观察区。
- 判断盈利是否大幅回撤。

### 5.2 最大可承受亏损

输入字段：

```text
max_loss_rate
```

默认值：

```text
8%
```

规则：

```text
holding_return <= -max_loss_rate -> 止损信号
```

### 5.3 目标止盈比例

输入字段：

```text
target_profit_rate
```

默认值：

```text
20%
```

规则：

```text
holding_return >= target_profit_rate
且 出现高位风险信号 -> 止盈或减仓信号
```

### 5.4 移动止盈线

第一版可先使用简化规则：

```text
近 20 日最高收盘价回撤超过 8% -> 盈利保护信号
```

未来可以升级为 ATR 动态止盈。

## 6. 卖出信号分层

### 6.1 硬止损信号

硬止损信号优先级最高。

触发条件：

- 持仓亏损超过最大可承受亏损。
- 放量跌破 20 日均线。
- 跌破年线且年线向下。
- 缠论跌破中枢下沿。

建议输出：

```text
建议止损：当前风险已超过继续观察范围，应优先控制亏损。
```

### 6.2 趋势破坏信号

触发条件：

- 收盘价跌破 20 日均线。
- 20 日均线向下。
- 年线下方运行。
- 缠论下降结构延续。

建议输出：

```text
建议减仓观察：趋势已经转弱，不适合继续重仓持有。
```

### 6.3 止盈信号

触发条件：

- 持仓收益超过目标止盈比例。
- 出现高位放量滞涨。
- 出现顶背驰候选。
- 近 20 日涨幅过高。
- 高位缩量离开中枢后动能不足。

建议输出：

```text
建议止盈或分批减仓：已有盈利且高位风险增加。
```

### 6.4 继续持有信号

触发条件：

- 收盘价仍在 20 日均线上方。
- 年线向上或价格站上年线。
- 回调缩量。
- 缠论结构未跌破中枢。
- 未触发止损或止盈风险。

建议输出：

```text
建议继续持有：趋势尚未破坏，但需要继续观察回调量能和关键支撑。
```

## 7. 建议的评分方式

卖出评分不要复用买入综合评分。建议新增：

```text
sell_risk_score: 0-100
```

含义：

```text
分数越高，越需要卖出或减仓。
```

建议规则：

```text
持仓亏损超过最大可承受亏损：+35
放量跌破 20 日均线：+25
跌破年线：+20
缠论跌破中枢候选：+20
缠论顶背驰候选：+18
高位放量滞涨：+15
20 日涨幅过高且量能异常：+10
收益超过目标止盈比例且出现风险信号：+15
收盘价站上 20 日均线且回调缩量：-10
年线向上且价格在年线上方：-8
```

映射建议：

```text
0-29：继续持有
30-49：减仓观察
50-69：建议止盈或止损
70-100：强风险，优先卖出控制风险
```

## 9. 前端设计

单股分析页可以增加“持仓卖出判断”区域。

输入项：

- 持仓成本价。
- 持仓数量，可选。
- 最大亏损比例，默认 8%。
- 目标止盈比例，默认 20%。

展示项：

- 当前持仓收益率。
- 卖出风险评分。
- 卖出建议。
- 关键支撑位：20 日均线、年线、中枢下沿。
- 止盈风险：高位放量滞涨、顶背驰候选、短期涨幅过高。
- 止损风险：亏损超阈值、放量跌破均线、跌破中枢。

前端不要只显示一句“卖出”。更适合展示：

```text
建议：减仓观察
原因：当前盈利 12.4%，但出现顶背驰候选并跌破中枢下沿。
关键位：若继续跌破 20 日均线，应进一步降低仓位。
```

## 10. CLI 设计

建议新增命令：

```bash
PYTHONPATH=src python3 -m cli sell 601988 --cost-price 5.20
```

可选参数：

```bash
--quantity 1000
--max-loss-rate 0.08
--target-profit-rate 0.20
--format json
```

输出内容：

- 股票代码。
- 股票名称。
- 当前收盘价。
- 持仓成本。
- 持仓收益率。
- 卖出风险评分。
- 卖出建议。
- 卖出理由。
- 继续观察点。

## 11. 最小可验收标准

第一版卖出模块完成时，应至少满足：

- 未输入持仓成本时，不输出止盈/止损建议，只输出技术风险提示。
- 输入持仓成本后，可以计算持仓收益率。
- 亏损超过最大可承受亏损时触发止损建议。
- 盈利较高且出现高位风险时触发止盈或减仓建议。
- 跌破 20 日均线、年线、中枢下沿时提高卖出风险评分。
- 回调缩量、趋势未破坏时允许继续持有。
- Web 和 CLI 都能展示卖出理由，而不是只给一个动作。
- 单元测试覆盖止损、止盈、减仓观察和继续持有四类结果。

## 12. 风险说明

卖出判断也不能保证卖在最高点。

系统应该明确提示：

- 卖出建议是风险控制辅助，不是收益承诺。
- 如果用户没有持仓成本，系统无法判断真实盈亏。
- 不同行业、不同波动率股票的止损和止盈比例不应完全相同。
- 后续接入 ATR 后，可以把固定止损升级为波动率自适应止损。
